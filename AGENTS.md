# AGENTS.md — Google Classroom 検索拡張機能

<!-- エージェントは本ガイドラインに厳密に従うこと。曖昧な点は作業を止め、人間に確認を依頼すること。 -->
<!-- 人間向けドキュメントは `README.md` を参照。 -->

## 0. 目的（変更不可）

- **高速かつ正確な横断検索** を Google Classroom Web に追加する
- **プライバシー重視**: ネットワーク送信なし、データは **IndexedDB のみ にローカル保存**
- UX 予算: 提案を **≤ 150 ms 以内**に表示（インデックス済み状態）、約 **1 万件までスケール**

## 1. やること / やらないこと

### やること

- 既存権限のみ使用: `https://classroom.google.com/*`, `storage`, `scripting`
- すべてのデータを **IndexedDB ストア内にのみローカル保存**: `gcsearch.documents`
- **段階的セレクタ**を実装: `プライマリ → フォールバック1 → フォールバック2`
- `教師` / `期限` / `添付ファイルタイプ` に **AND フィルタ**を適用
- 初期インデックスに**アイドル時間チャンキング**を使用（`requestIdleCallback`、約 50 ms スライス）
- **差分再インデックス**に MutationObserver を使用

### やらないこと

- **新規ネットワーク呼び出し禁止**（CDN/アナリティクス/テレメトリを含む）
- 権限を拡張したり、ホストドメインを追加しない
- **可視 DOM** 以外のプライベートフィールドをスクレイプしない
- このファイルを編集せずに検索の重み付け/閾値を変更しない
- タスクごとにメインスレッドを 50 ms を超えてブロックしない

## 2. 実行環境とコマンド

- **Node**: v20 LTS / **パッケージマネージャ**: npm
- インストール: `npm i`
- 開発（ウォッチモード）: `npm run dev`（任意）
- ビルド（MV3）: `npm run build` → `dist/` を出力（任意、アンパックロード も可）
- テスト: `npm test`（任意）
- リント: `npm run lint`

## 3. データ契約（IndexedDB）

ストア名: `gcsearch.documents`  
型（概念）:

```javascript
type Kind = 'assignment'|'material'|'post'|'comment';
type Att = 'docs'|'sheets'|'slides'|'pdf'|'link'|'other';

interface Attachment { kind: Att; title: string; url: string; }
interface Document {
  id: string; url: string; type: Kind;
  className: string; teachers: string[]; topic?: string;
  title: string; text: string; due?: number;
  attachments: Attachment[]; updatedAt: number;
}
```

- **GC ポリシー**: 最新 **N = 120 日** を保持、ピン留めされていない場合は古いものをロールオフ
- **PII 拡張なし**: Classroom UI で既に可視な情報のみ保持

## 4. 検索仕様（Fuse.js）

- **キーと重み**: `title^3`, `topic^2`, `text`, `className`, `teachers`, `attachments.title`
- **設定**: `threshold: 0.30`, `ignoreLocation: true`, `minMatchCharLength: 2`
- **日本語処理**: プレフィックス優先の **サジェスト辞書** + **2/3-gram トークン化**
- **結果**: 上位 **20 件**、マッチしたトークンの **ハイライト** スパン付き

## 5. DOM 抽出に関する注意

- **対象ページ**: クラス一覧 / ストリーム / 課題 / 教材 / コメント
- **セレクタ段階の例**:
  - タイトル: `h1[aria-level="1"]` → `[data-title]` → `.YVj4Ic`
  - 期限: `[data-due]` → aria テキスト → プレーンテキストパターン `期限:`
- **プライマリセレクタの失敗率が 30% を超える** 場合、DOM を **劣化** として マーク
- ページテーマを尊重し、テーマ切り替えを監視してハイライトを再描画

## 6. インデックスと性能

- **初期インデックス**: `requestIdleCallback` 経由の **アイドル時間チャンキング**（スライス約 50 ms）
- **差分**: **MutationObserver パッチ**（デバウンス済み）
- **予算（インデックス済み状態）**: 1 トークンクエリで最初の提案まで **50～150 ms**
- 同期的な重い処理ループを避ける、マイクロタスクに処理を分割

## 7. フィルタ

- **すべてのフィルタに AND セマンティクス** を適用
- `期限` は `<=`, `>=`, および範囲をサポート、**エポック ms** として比較（包括的な境界）
- `添付ファイルタイプ` は **MIME / URL パターン / 拡張子** の順に決定

## 8. API フォールバック（OAuth、読み取り専用）

トリガー条件:
  1. セレクタ命中率が **> 30%** 低下、または
  2. データセットが **> 30k** で遅延が継続する場合

フロー: 同意モーダル → OAuth（Classroom API、**読み取り専用**）→ **同じスキーマ** にマップ  
ストレージは **IndexedDB のまま**、**テレメトリなし** を追加

## 9. 受け入れ基準（必須）

1. 1 トークン → 提案を **≤ 150 ms** で表示
2. 教師 / 期限 / 添付ファイル **AND フィルタ** が正確
3. 添付ファイル **タイトル/ファイル名** が検索可能
4. ダーク/ライトの両方が読みやすい（コントラスト/ハイライト AA）
5. **DevTools でネットワーク通信なし**（API モードを除く）
6. DOM 破綻 → **API プロンプト表示** → 同意 → 検索機能が動作

## 10. セキュリティとプライバシー

- **既定**: アナリティクスなし、リモート通信なし。オプションページは **再インデックス / データ消去 / モード切り替え** を提供
- 権限やデータ保持ポリシーの変更 → `AGENTS.md` + `docs/security.md` を更新して PR を開く
